## 路由设计

使用 HashRouter（Capacitor file:// 协议兼容）。

```
/                                  → Home
/series/:seriesId                  → BookList
/book/:seriesId/:bookIdx           → ChapterList
/reader/:seriesId/:bookIdx/:chapterIdx → Reader
/search                            → Search
/profile                           → Profile
```

Reader 页可携带 `?from=search` 参数，标记为搜索跳转，跳过阅读计时和历史记录。

## 数据架构

### 数据文件

```
public/data/
├── index.json            # 目录索引，启动时加载 (~219KB)
│   { series: [{ id, label, books: [{ title, file, chapterCount,
│       chapters: [{ title, url }] }] }] }
├── search-index.json     # 搜索索引，首次搜索时懒加载 (~5.4MB)
│   [{ s: seriesId, sl: seriesLabel, b: bookIdx, bt: bookTitle,
│      c: chapterIdx, ct: chapterTitle, t: snippet(1500字) }]
└── books/                # 62 本书，按需加载 (~300-400KB/本)
    s1_b灵修指微.json
    s1_b十字架的道.json
    ...
```

### 加载策略

```
启动
 ├─ fetchIndex()        → 加载 index.json → 内存缓存
 └─ 首页渲染

打开书籍
 ├─ fetchBook(sid, bidx) → 加载 books/xxx.json → 内存缓存 (Map)
 └─ Reader 渲染章节内容

打开搜索
 └─ 首次 focus → 加载 search-index.json → 内存缓存
```

### 缓存策略

- `indexCache`: 模块级变量，生命周期 = 页面存活时间
- `bookCache`: `Map<string, BookData>`，无限增长（单次会话内浏览书籍有限）
- 不设缓存过期，刷新页面即清空

## 状态管理

### settings store (Zustand + persist)

```typescript
{
  theme: 'light' | 'dark' | 'sepia'      // 主题
  fontSize: 'small' | 'medium' | 'large' | 'xlarge'  // 字号
  progress: {                              // 阅读进度
    [key: `${seriesId}-${bookIdx}`]: {
      seriesId, bookIdx, chapterIdx, scrollPos, updatedAt
    }
  }
  lastRead: { seriesId, bookIdx, chapterIdx } | null
}
```

持久化到 localStorage key: `niwj-reader-settings`

### reading store (Zustand + persist)

```typescript
{
  totalMinutes: number                    // 累计阅读分钟
  chaptersRead: { [key]: true }          // 已读章节集合
  dailyActivity: [{ date, minutes }]     // 每日阅读分钟 (90天)
  history: HistoryEntry[]                // 最近 50 条阅读记录
}
```

持久化到 localStorage key: `niwj-reader-reading`

### 阅读计时机制

```
Reader mount
 └─ setInterval(1s)
     └─ document.visibilityState === 'visible' ? seconds++ : skip
     └─ 每 30s 批量写入 addReadingTime(30)

Reader unmount
 └─ 结算剩余秒数 addReadingTime(remaining)
```

- `visibilitychange` 事件: 页面切换到后台自动暂停计时
- `?from=search` 参数: 搜索跳转不启动计时器

## 主题系统

### 实现方式

CSS Variables 三级主题：

```css
:root { --bg, --surface, --text, --text-secondary, --accent, ... }
[data-theme='dark'] { ... }
[data-theme='sepia'] { ... }
```

### 同步机制

1. **初始加载**: `index.html` 内联脚本读取 localStorage，在 React 挂载前设置 `data-theme`，避免闪烁
2. **运行时切换**: `ThemeSync` 组件（App 层）监听 `theme` state，同步更新 `<html data-theme>` 和 `<meta name="theme-color">`
3. **状态栏**: 通过 `theme-color` meta 标签同步 Android/iOS 状态栏颜色

## 搜索架构

### 三层内部导航

```
L1 (搜索首页)
 ├─ 输入关键词 → allResults (useMemo, 上限 200 条)
 ├─ 按 series→book 分组展示
 ├─ 点章节 → goL3()
 └─ 点「查看全部」→ goL2()

L2 (书卷筛选)
 ├─ l2Results: 单书的匹配章节
 └─ 浏览模式 (无关键词): 显示该书全部章节

L3 (文本定位)
 ├─ fetchBook() 加载完整章节内容
 ├─ HighlightedText: 关键词正则分割 + <mark> 高亮
 ├─ 双 rAF 确保 DOM 就绪后自动滚动到首处
 └─ 下一处/上一处: markRefs 数组索引跳转
```

### 竞态保护

```typescript
const goL3ReqId = useRef(0)
const goL3 = async (...) => {
  const reqId = ++goL3ReqId.current
  const book = await fetchBook(...)
  if (reqId !== goL3ReqId.current) return  // 丢弃过期请求
  setL3Content(...)
}
```

### 系统返回键

```typescript
// 仅在首次离开 L1 时 pushState，避免历史记录膨胀
useEffect(() => {
  if (level !== 'L1' && !pushedForLevel.current) {
    window.history.pushState(...)
    pushedForLevel.current = true
  }
}, [level])

// popstate + Capacitor backButton 双监听
window.addEventListener('popstate', handleBack)
App.addListener('backButton', handleBack)
```

## 构建与部署

### 本地开发

```bash
npm run dev           # Vite dev server → http://localhost:5173
```

### H5 生产构建

```bash
npm run build         # → dist/ (JS + CSS + data 文件)
```

### Android APK

```bash
npm run build
npx cap sync android
cd android && ./gradlew assembleDebug     # → app-debug.apk (~13MB)
```

### CI (GitHub Actions)

Push main → `.github/workflows/build-apk.yml`:
1. Node 22 + npm ci
2. Vite build
3. Capacitor sync
4. JDK 17 + Android SDK 34 + Gradle assembleDebug
5. Upload APK artifact (保留 30 天)

## iOS 预留

架构已支持 iOS，需 macOS 环境：

```bash
npx cap add ios
npx cap open ios     # Xcode 配置签名 → Archive
```
